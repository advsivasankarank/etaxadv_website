# e-TDS QC Tool Deployment Guide

## Target Path

- internal app URL: `/fintech/etds-qc/`
- legacy redirect kept at `/fintech/etds-qc.php`

## Runtime Requirements

- PHP 8.2 or newer
- extensions: `zip`, `json`, `mbstring`, `gd`, `SimpleXML`, `fileinfo`
- web server with PHP enabled
- writable storage path: `storage/etds-qc/`
- optional OCR runtime: `C:\Program Files\Tesseract-OCR\tesseract.exe`

## Files To Deploy

- `fintech/etds-qc/`
- `fintech/etds-qc.php`
- `fintech-tools.php`
- `includes/header.php`
- `storage/etds-qc/.htaccess`
- `storage/etds-qc/config.json`
- `storage/etds-qc/users.json`

## Directory Permissions

Grant web server write permission to:

- `storage/etds-qc/`
- `storage/etds-qc/sessions/`

Keep web access blocked for:

- JSON files
- logs
- generated exports
- uploaded source files

## First-Time Setup

1. Deploy the files to the website root.
2. Confirm `storage/etds-qc/` exists and is writable.
3. Open `/fintech/etds-qc/?view=login`.
4. Sign in with the bootstrap admin account.
5. Change the password hash inside `storage/etds-qc/users.json`.
6. Create a sample QC session and verify upload, validation, reconciliation, and export.

## Password Rotation

Update `password_hash` in `storage/etds-qc/users.json` using a value generated by:

```bash
php -r "echo password_hash('YourStrongPassword', PASSWORD_DEFAULT), PHP_EOL;"
```

## OCR Support

Image OCR is enabled when Tesseract is available at:

`C:\Program Files\Tesseract-OCR\tesseract.exe`

If Tesseract is not available, image uploads remain stored and previewable, but OCR extraction falls back to review-only mode.

## PDF Support

PDFs use an internal text-extraction fallback for text-based PDFs.

Scanned PDFs that do not expose extractable text should be treated as manual-review or OCR-preparation inputs.

## Purge Behaviour

- purge threshold comes from `storage/etds-qc/config.json`
- default `purge_after_days` is `7`
- archived or downloaded sessions older than the threshold are auto-purged on later app bootstrap
- purge retains session metadata and audit history

## Smoke Checklist

- login succeeds
- session creation succeeds
- CSV upload extracts records
- XLSX upload extracts records
- image upload stores OCR text excerpt when available
- challan reconciliation computes totals
- ready session generates `.xlsx` output

## Rollback

To roll back the feature:

1. restore previous copies of `fintech-tools.php`, `fintech/etds-qc.php`, and `includes/header.php`
2. remove `fintech/etds-qc/`
3. archive or remove `storage/etds-qc/` after backing up any live QC sessions
